# 第1部 Speculation Game の構造と数式

## 1.1 モデルの全体像（一枚スケッチ）

```
┌──────────────────────────────────────────┐
│  N 人のトレーダー（i = 1, ..., N）          │
│                                              │
│  各 i は持ち物：                              │
│   - 戦略テーブル s_{i,j,μ} ∈ {-1, 0, +1}    │
│       j: 戦略番号 (j = 1, ..., S)           │
│       μ: 履歴 index (μ = 0, ..., K-1)       │
│       K = 5^M                               │
│   - 認知利得 G_{i,j} ∈ ℤ                    │
│   - 口座残高 w_i ∈ ℤ_{≥0}                   │
│   - active strategy index j*_i              │
│   - ポジション pos_i ∈ {-1, 0, +1}          │
└──────────────────────────────────────────┘
                ↓
        全員の注文を集計
                ↓
┌──────────────────────────────────────────┐
│  価格更新（需給集計型）                       │
│   Δp = (1/N) Σ_i a_i · q_i                  │
│   p ← p + Δp                                │
│   h = quantize(Δp, C) ∈ {-2,-1,0,+1,+2}     │
│   P ← P + h                                 │
│   μ ← shift_in(μ, h+2, M)                   │
└──────────────────────────────────────────┘
                ↓
        各 i の close を処理
                ↓
┌──────────────────────────────────────────┐
│  破産判定 w_i < B → 入れ替え                  │
└──────────────────────────────────────────┘
```

## 1.2 3層構造（口述で必ず言える形）

**第1層：戦略の層**
- 各エージェントは S 枚の戦略テーブルを持つ
- 各戦略は「過去 M 期間の価格量子化履歴 μ ∈ [0, 5^M)」に対して `{買う=+1, 売る=-1, 何もしない=0}` を返す関数
- 戦略の評価は「実損益 + 仮想損益 G_{i,j}」で行う
- 評価が高い戦略を active として選択（同点なら最近の選択を維持）

**第2層：口座残高の層**
- 各エージェントは口座残高 w_i を持つ
- 発注量は q_i = ⌊w_i / B⌋（B は1単元の取引コスト、YH006 では B=9）
- round-trip（open → close）で利得 ΔG が確定し、w に反映される

**第3層：価格形成の層**
- 全員の注文を集計：D = Σ a_i · q_i
- 価格更新：Δp = D/N
- 量子化：h = quantize(Δp, C) ∈ {-2, -1, 0, +1, +2}
- 認知価格：P ← P + h（戦略のための「価格信号」）
- 履歴更新：μ ← shift_in(μ, h+2, M)

## 1.3 戦略テーブルの数学

### 1.3.1 履歴 μ の表現

M 期間の価格変化を5値量子化したものを、5進数として整数 μ ∈ [0, 5^M) で表現する。

例えば M=3 で、過去3期の量子化が (h_{-3}, h_{-2}, h_{-1}) = (+2, -1, 0) の場合、
これを (h+2 を加えて) (4, 1, 2) とし、
μ = 4·5² + 1·5¹ + 2·5⁰ = 107

K = 5^M が「履歴 index の取りうる値の数」。M=5 なら K = 3125。

### 1.3.2 戦略テーブル

各エージェント i、各戦略 j について：
```
s_{i, j} : [0, K) → {-1, 0, +1}
```
すなわち長さ K の int8 配列。初期化はランダム（一様分布で {-1, 0, +1} から選ぶ）。

### 1.3.3 戦略の評価 G

G_{i,j} は戦略 j の累積利得。
- **実利得**：自分が active strategy として使ったときに発生した round-trip の ΔG を足す
- **仮想利得**：使っていない戦略についても「もし使っていたら」という仮想ポジションを保持し、close 相当のタイミングで ΔG_virtual を G に加える

active の評価が他より低くなったら、最大値の戦略に乗り換える。

### 1.3.4 履歴の更新 shift_in

```python
def shift_in(mu: int, new_value: int, M: int) -> int:
    """5進シフト。new_value ∈ [0, 5) を末尾に入れる。"""
    K = 5 ** M
    return ((mu * 5) + new_value) % K
```

このシフトで、最古の桁が捨てられて新しい桁が入る。

## 1.4 富のダイナミクス：q = ⌊w/B⌋

### 1.4.1 発注量の決め方

エージェント i が open するとき：
- 発注量 q_i = ⌊w_i / B⌋ （口座残高を1単元コスト B で割った整数）
- 注文の符号 a_i ∈ {-1, +1}（買いか売り）

YH006 では B=9。なので w=50 なら q=5。w=90 なら q=10。

### 1.4.2 close 時の損益

open 時に記録した：
- entry_price = P（認知価格）
- entry_action = a（買いか売り）
- entry_quantity = q

close 時：
- 現在の認知価格 P_now
- ΔG = entry_action × (P_now - entry_price)
- 口座残高更新：w ← w + ΔG × entry_quantity

ポイント：**ΔG は1単元あたりの利得**で、口座残高には entry_quantity 倍されて反映される。だから q が大きいエージェントほど、同じ ΔG でも口座残高への影響が大きい。これが「お金が動きを変える」フィードバックの数学的中身。

### 1.4.3 富の分布が冪乗則に向かう理由（直感）

- 当たり戦略を持ったエージェントは w が増え、次は大きな q で張れる
- 大きな q で当たればさらに w が増える（multiplicative growth）
- multiplicative growth + 退場と再生による境界条件 → log-normal もしくは Pareto-like な定常分布

これが Katahira 2019 の主張：「ボラティリティが固まる現象の起源は、この富の不均一性が q の不均一性に伝わる経路にある」。

## 1.5 価格形成：需給集計型

### 1.5.1 effective action の決定

各エージェント i について、推奨 rec ∈ {-1, 0, +1}（戦略テーブルから引いた値）とポジション pos に応じて：

| pos | rec | 結果 | a_i | q_i |
|---|---|---|---|---|
| 0 | 0 | idle | 0 | 0 |
| 0 | ±1 | open | rec | ⌊w/B⌋ |
| ±1 | 0 | active_hold | 0 | 0 |
| ±1 | pos | passive_hold | 0 | 0 |
| ±1 | -pos | close | rec | entry_quantity |

つまり、open と close のときだけ実際に注文が市場に出る。hold は既ポジションを維持する論理状態。

### 1.5.2 価格更新

```
D = Σ_i a_i · q_i
Δp = D / N
p ← p + Δp
```

ここでの p は連続値（float）。Δp も連続値。

## 1.6 量子化と認知価格

実価格 p は連続だが、戦略の入力に使う「履歴」は離散の量子化値が必要。

### 1.6.1 量子化関数 quantize

```python
def quantize(dp: float, C: float) -> int:
    """dp ∈ ℝ を5値量子化 ∈ {-2, -1, 0, +1, +2}。
    
    C は「±1 と ±2 の境界」を決めるスケールパラメータ。
    """
    if dp > C:
        return 2
    elif dp > 0:
        return 1
    elif dp < -C:
        return -2
    elif dp < 0:
        return -1
    else:
        return 0
```

YH006 では C=3.0 を基本に使う。これは「典型的な Δp の桁感に合わせて、5値カテゴリが均等に出るように」調整したもの。

### 1.6.2 認知価格 P

```
h = quantize(Δp, C)
P ← P + h
```

P は整数の累積。エージェントの戦略は P と entry_price の差で損益を計算する。これは「価格そのものの細かい変化ではなく、量子化された大きさで損益を測る」というモデル化。

## 1.7 戦略レビューと virtual 更新

### 1.7.1 active 戦略の見直し

close が起きた直後に、自分が今 active にしている戦略 j*  が G の最大値の集合 best = argmax G_{i,j} に含まれているかを確認。

- j*  ∈ best → 続行
- j*  ∉ best → best からランダムに新しい戦略を選んで切り替える
  - 切り替えた新戦略の virtual_position をクリア

### 1.7.2 仮想ポジション（virtual）

active でない戦略 j について、「もし使っていたら」という仮想ポジションを保持する：
- 各 step で戦略 j の推奨 rec_j を読む
- vpos_j == 0 で rec_j != 0 → 仮想 open（vpos_j = rec_j, virtual_entry_price = P, virtual_entry_action = rec_j）
- vpos_j != 0 で rec_j == -vpos_j → 仮想 close（ΔG_v を G_{i,j} に加える、virtual をクリア）
- それ以外 → hold（virtual の状態維持）

これにより、使われていない戦略も「もし使っていたらどうだったか」が継続的に評価され、active の交代候補として比較可能になる。

## 1.8 破産退場と新規参入

close 直後の破産判定：
- w < B（1単元すら買えない）→ 退場
- 退場すると、新しいエージェントが置き換わる：
  - 戦略テーブルをランダムに再生成
  - w = B + U[0, 100) でランダムな初期残高
  - active_idx もランダム
  - position, virtual すべてクリア
  - G もゼロクリア

この入れ替えが、富の分布の定常状態を維持するエンジン。退場と再生を止めると、富の分布は崩壊する。

|---|---|---|
| N | エージェント数 | 100 |
| M | 履歴の長さ | 5 |
| S | 1人あたり戦略テーブル数 | 2 |
| K | 履歴 index の上限 = 5^M | 3125 |
| B | 1単元の取引コスト | 9 |

## 1.10 SG vs Minority Game（暗記表）

| | Minority Game | Speculation Game |
|---|---|---|
| 勝ち負け | 少数派が勝つ（人工的） | 価格変動の差分が損益（市場的） |
| 戦略出力 | {-1, +1} の2択 | {-1, 0, +1} の3択 |
| 富 | 仮想点数のみ | 実残高、発注量に効く |
| 価格 | 多数派決定で反転 | 需給集計で動く |
| 退場 | なし（人数固定） | 破産で入れ替え |

**最重要の違い**：SG は「w → q → 価格」の経路を持つ。MG は持たない。

# 第2部 Python 実装の逐行理解

ここでは `packages/abm_models/abm_models/sg/reference.py` のメインループを逐行で読み解く。「自分で書ける」レベルを目指す。

## 2.1 初期化（RNG 消費順序が肝）

```python
rng = np.random.default_rng(seed)

# ステップ1: 戦略テーブル（N × S × K の3次元配列）
all_strategies = rng.choice([-1, 0, 1], size=(N, S, K)).astype(np.int8)

# ステップ2: 初期残高の乱数オフセット
init_u100 = rng.integers(0, 100, size=N)  # 各エージェントに U[0, 100)

# ステップ3: 各エージェントの初期 active 戦略
init_active = rng.integers(0, S, size=N)

# ステップ4: 初期履歴 index
mu0 = int(rng.integers(0, K))
```

**なぜこの順序が重要か**：bit-parity（参照実装とベクトル化実装で1ビットも違わない再現）を担保するため。同じ seed なら、同じ順序で乱数を消費すれば同じ結果になる。

各エージェントの初期残高は `w = B + init_u100[i]` で計算（B=9 + U[0, 100) なので 9 ≤ w ≤ 108）。

## 2.2 メインループの9段階

`for t in range(T):` の中身を、9段階に分解して読む。

### 段階1: μ のスナップショット

```python
mu_t = market.mu
```
このステップで使う履歴 index を固定する。後で更新されるので、ここで取っておく必要がある。

### 段階2: 各エージェントの推奨アクション

```python
recs = np.zeros(N, dtype=np.int8)
for i in range(N):
    a = agents[i]
    recs[i] = int(a.strategies[a.active_idx, mu_t])
```
各エージェントの active 戦略テーブルから、現在の履歴 mu_t に対応する推奨アクションを取り出す。

### 段階3: effective action 表の適用

```python
for i in range(N):
    a = agents[i]
    rec = int(recs[i])
    pos = a.position
    if pos == 0:
        if rec == 0:
            kinds[i] = "idle"
        else:
            effective[i] = rec
            quantity[i] = a.w // B
            kinds[i] = "open"
    else:  # pos != 0
        if rec == 0:
            kinds[i] = "active_hold"
        elif rec == pos:
            kinds[i] = "passive_hold"
        else:  # rec == -pos
            effective[i] = rec
            quantity[i] = a.entry_quantity
            kinds[i] = "close"
```
これが 1.5.1 の表をそのままコード化したもの。注意：close 時の quantity は entry_quantity であって `w // B` ではない（open 時の枚数を閉じるので）。

### 段階4: 需給集計と価格更新

```python
D = int((effective * quantity).sum())
dp = D / N
market.p += dp
```

### 段階5: 量子化と履歴更新

```python
h = quantize_price_change(dp, C)
market.P += h
if history_mode == "endogenous":
    market.mu = shift_in(market.mu, h + 2, M)
else:  # 'exogenous'
    market.mu = int(rng.integers(0, K))
```
exogenous モードは「履歴を外から乱数で投入する」モード。ベースラインや感度分析で使う。

### 段階6: close 実行（状態クリア前にラウンドトリップ記録を append）

```python
for i in range(N):
    if kinds[i] == "close":
        a = agents[i]
        dG = int(a.entry_action) * (market.P - int(a.entry_price))
        # ラウンドトリップを記録
        rt_agent_idx.append(i)
        rt_open_t.append(int(a.entry_step))
        rt_close_t.append(t)
        rt_entry_action.append(int(a.entry_action))
        rt_entry_quantity.append(int(a.entry_quantity))
        rt_delta_G.append(dG)
        # 状態更新
        a.G[a.active_idx] += dG
        a.w += dG * int(a.entry_quantity)
        # 状態クリア
        a.position = 0
        a.entry_price = 0
        a.entry_action = 0
        a.entry_quantity = 0
        a.entry_step = 0
```

**ここで dG の符号に注意**：a.entry_action（買いなら +1, 売りなら -1）と価格変化（P - entry_price）の積。買い建てした人にとっては P が上がれば利得、売り建てした人にとっては P が下がれば利得。これが両方とも entry_action × (P_now - entry_price) で表現される。

### 段階7: open 実行

```python
for i in range(N):
    if kinds[i] == "open":
        a = agents[i]
        a.position = int(effective[i])
        a.entry_price = market.P
        a.entry_action = int(effective[i])
        a.entry_quantity = int(quantity[i])

```

### 段階8: virtual 戦略の更新

```python
for i in range(N):
    a = agents[i]
    for j in range(S):
        if j == a.active_idx:
            continue
        rec_ij = int(a.strategies[j, mu_t])
        vpos = int(a.virtual_positions[j])
        if vpos == 0:
            if rec_ij != 0:
                a.virtual_positions[j] = rec_ij
                a.virtual_entry_prices[j] = market.P
                a.virtual_entry_actions[j] = rec_ij
        else:
            if rec_ij == -vpos:  # 仮想 close
                dG_v = int(a.virtual_entry_actions[j]) * (market.P - int(a.virtual_entry_prices[j]))
                a.G[j] += dG_v
```

## 3.1 2×2 設計の論理

問い：「SG の口座残高ダイナミクスは、需給集計型から LOB 型に世界を変えたときに、初期富分布への感度がどう変化するか？」

これを2軸で分解：
- **世界軸**：需給集計型（YH005 オリジナル）vs LOB 型（PAMS による拡張）
- **初期富分布軸**：一様 U[B, B+100) vs Pareto α=1.5

| | 一様初期残高 | Pareto α=1.5 初期残高 |
|---|---|---|
| 需給集計型 (aggregate) | **C0u** | **C0p** |
| LOB 型 (PAMS) | **C2** | **C3** (主実験) |

加えて、**C1**：LOB 側で SG エージェントなし（FCN のみ、純粋な流動性ベースライン）。N=100、c_ticks ≈ 28.0 tick で統一。

## 3.2 各条件の意味

- **C0u**：原典の SG をそのまま。基準点。
- **C0p**：原典 SG に、初期富分布だけを Pareto に置き換え。「もし初期から富が偏っていたら、SG はどう振る舞うか」を見る。SG は通常 substitute（破産退場）で初期分布を「洗い流す」と予測される。
- **C2**：SG の戦略決定だけを保ったまま、市場世界を LOB に置き換え。「LOB 摩擦下で SG はどう振る舞うか」。
- **C3**：C2 に加えて初期富分布も Pareto。これが主実験。「LOB で初期分布の偏りが残るか、洗い流されるか」が核心。

## 3.3 N=100 統一の理由

旧来の比較では aggregate 側を N=1000、LOB 側を N=100 にしていた。これだと「N の効果」と「世界の効果」が混ざる（confound）。

例えば Hill 指数の α は N に強く依存する（N が小さいとテール推定が不安定）。N=1000 → 100 で α の見かけが変わるが、これが「LOB の効果」と区別できない。

**N=100 で全条件を揃えることで、N 効果を消し、世界効果だけを純粋に取り出せる。**

## 3.4 c_ticks の役割

LOB の tick scale（最小値刻み）を、aggregate 世界の典型的な |Δp| に合わせるための較正値。

具体的には：C1（LOB 流動性のみ）の median |Δmid| を測り、その 3 倍を c_ticks とする。YH006 では c_ticks ≈ 28.0 tick が採用された。

**なぜ較正が要るか**：LOB の tick が小さすぎると価格変動の解像度が過剰、大きすぎると粗すぎる。aggregate 世界での Δp スケールに合わせることで、両世界を「同じ価格変化幅で見られる」状態にする。

## 3.5 PAMS と MMFCNAgent

PAMS（Hirano-Izumi 2023）は Python の金融 ABM フレームワーク。Limit Order Book を実装している。

- **FCNAgent**: Fundamental-Chartist-Noise の3要素を組み合わせた基本トレーダー。流動性供給役。
- **MMFCNAgent**: PAMS 0.2.2 の `FCNAgent.submit_orders_by_market` が order_volume を1にハードコードしている問題への対応として、order_volume=30 で出させる workaround。YH006 では流動性層として併走。

## 4.3 A1 ablation（q を全員一律に固定）

通常 SG では q_i = ⌊w_i / B⌋。これを `q = q_const`（全エージェントで同じ値）に置き換える。

**q_const の決め方**（Appendix A.1）：
1. C3 setup の pilot 5 trial を seed 9001..9005 で実行
2. 各 trial で median_{t, i}(q_i(t)) を計算（全時刻・全エージェントの q の中央値）
3. 5 trial の median のさらに median を q_const とする
4. q_const ≥ 2 が最低要件

**なぜ中央値ベースか**：Pareto 重尾下で平均値は不安定（外れ値に引きずられる）。中央値は典型エージェントの活動量を保存する。

**ablation のロジック**：仮説 A が正しければ、q を全員一律にすると「富のヘテロ性が q のヘテロ性に伝わる」経路が物理的に切れるので、interaction が消えるはず。

## 4.4 A3 ablation（寿命上限）

**τ_max の決め方**：
1. C3 pilot で agent ごとの lifetime（滞在 step 数）の中央値 L_50 を測る
2. τ_max = round(0.5 × L_50) を初期値
3. 強制 substitute が trial 全体で 50 件以上発生するように調整

**ablation のロジック**：仮説 A の chain の中で、「lifetime 延長 → 初期分布 persist」の部分を切る。これで A の chain がどこで効いているかを解像度高く見る。

## 4.5 主指標と補助指標

### 4.5.1 主指標（interaction の robustness）

corr(|ΔG|, h) を3つの相関係数で並走計算：
- **ρ_P (Pearson)** — Phase 1 互換、artifact 検出用
- **ρ_S (Spearman)** — 主指標

### 4.5.2 Funnel 構造の直接計測

### 4.6.2 信頼区間

trial-level の Spearman ρ から **percentile bootstrap (resample = 10,000)** で 95% CI を構成。

**なぜ Fisher-z を使わないか**：Fisher-z 変換（z = arctanh(ρ)）は ρ の標本分布を近似正規にする変換で、Pearson の信頼区間に標準的。しかし重尾分布のもとでは Fisher-z の近似が破綻する。Bootstrap は分布を仮定しないので、重尾でも頑健。

### 4.6.3 条件間比較

- **2 条件比較**（例: C3 vs C3_A1）：Mann-Whitney U test（両側、各 N=100）
- **interaction の有意性**：差分の差分の bootstrap 経験分布から 95% CI を取る

### 4.6.4 多重比較補正

主指標 1 + ablation 比較 2（A1, A3）+ funnel 直接指標 1（bin variance）で **4 系統**。Bonferroni 補正で α = 0.05 / 4 = 0.0125 を主検定の閾値。

# 第5部 統計手法の選択論理（自分で発想できるように）

「なぜその手法か」を、ゼロから自分で考えられるレベルで整理する。

## 5.1 相関係数：Pearson, Spearman, Kendall

### 5.1.1 Pearson の定義と前提

```
ρ_P = Cov(X, Y) / (σ_X · σ_Y)
```

**ρ_P が well-defined であるための条件**：
- σ_X, σ_Y が有限（分散が定義できる）
- これだけ。正規性は不要

**標準的な推論（CI、検定）が valid であるための追加条件**：
- 関係が線形（非線形な単調関係は過小評価される）
- 標準誤差の計算には4次モーメント（kurtosis）が有限であることが効く
- Fisher-z 信頼区間は近似的に二変量正規を仮定

**テール指数 α とモーメントの関係（重要）**：

重尾分布 CCDF が P(|X| > x) ∝ x^{-α} に従うとき、モーメントの存在条件は α に依存する：

- **α ≤ 1**：平均すら発散
- **1 < α ≤ 2**：平均は有限、**分散は無限**。Mandelbrot の stable Paretian 仮説はこの領域で立てられた
- **2 < α ≤ 4**：分散は有限だが **kurtosis（4次モーメント）が無限**
- **α > 4**：通常の漸近論が使える

実際の金融市場では **α ≈ 3 程度**（"inverse cubic law"、Gopikrishnan et al. 1998 など）が観測されることが多い。これは「分散は有限だが kurtosis は無限」の領域。

**この α ≈ 3 領域で Pearson に何が起きるか**：

- 分散自体は有限なので、標本相関 ρ̂ は真値に収束する（一致性はある）
- しかし収束は遅い：Var(ρ̂) の漸近式が kurtosis に依存し、kurtosis が無限なので CLT の標準誤差公式が破綻する
- 単一の極端な観測が標本分散を大きく動かし、ρ̂ を 1〜数試行の間で大きく揺らす
- Fisher-z による信頼区間は kurtosis 有限を暗黙に仮定しているので不正確

YH006 Phase 1 で Pearson −0.27 を観察したが、これが「本物の単調関係」なのか「数個の極端な ΔG に引きずられた標本誤差」なのか、Pearson だけでは（しかも 1 trial では）判別できない。だから Spearman を主指標に置く。

### 5.1.2 Spearman の発想

「線形性を仮定したくない、外れ値の影響を消したい」→ **値そのものではなく順位を使う**。

```python
def spearman(x, y):
    rank_x = rank(x)  # 1, 2, 3, ... に変換
    rank_y = rank(y)
    return pearson(rank_x, rank_y)
```

順位は単調変換に対して不変（log を取っても順位は変わらない）。外れ値も、順位にしてしまえば「最大値の順位は n」というだけで、値そのものの影響を受けない。

**Spearman が測るもの**：単調関係の強さ。「X が増えると Y も増える傾向」を、線形でなくても検出できる。

### 5.1.3 Kendall の発想

「順位の差を集計する Spearman より、もっと外れ値に頑健な方法はないか」→ **ペアの順序の一致を数える**。

```
τ = (C - D) / (n(n-1)/2)
C: concordant pairs の数（X_i < X_j かつ Y_i < Y_j のペア）
D: discordant pairs の数（X_i < X_j かつ Y_i > Y_j のペア）
```

ペアごとに「順序が一致しているか反対か」だけを見るので、極端な値の影響を受けにくい。計算量は O(n²) で重いが、最頑健。

### 5.1.4 なぜ3つ全部使うか

それぞれが違う種類のロバストネスを持つ：
- Pearson 単独：線形性と有限分散を仮定。α ≤ 2 では分散自体が無限で意味を失う。α ≈ 3 では分散有限だが kurtosis 無限で、標準誤差・CI が信用できない
- Spearman 単独：単調性を仮定。順位タイがあると効率が落ちる
- Kendall 単独：最頑健だが計算が重く、効果サイズの解釈が違う

3つの符号と桁が一致すれば「実際に何か単調な関係がある」ことの強い証拠になる。3つが食い違ったら、どれかが artifact を拾っている。

## 5.2 Hill 推定量（テール指数 α）

### 5.2.1 問題設定

重尾分布 P(|X| > x) ∝ x^{-α} の α を推定したい。

### 5.2.2 直感的な発想

「対数空間で見れば、テールは直線になる」：
```
log P(|X| > x) = log C - α · log x
```

ということは、テールのデータ点を log-log プロットして、その傾きが −α。

### 5.2.3 Hill の式

サンプルを降順に並べて X_{(1)} ≥ X_{(2)} ≥ ... ≥ X_{(n)}。上位 k 個を「テール」として：

```
α̂ = 1 / [(1/k) Σ_{i=1}^{k} (log X_{(i)} - log X_{(k+1)})]
```

これは MLE（最尤推定）で、上位 k 個が α-Pareto に従うと仮定したときの α の最尤推定値。

### 5.2.4 k の選び方

- k が小さすぎる：分散が大きい
- k が大きすぎる：テール以外のデータが混じる（バイアス）
- 典型的に k = √n か、xmin = 90 percentile を境界に取る

YH006 では xmin = p90 を採用。「上位 10% をテールとして α を推定」。これで n_tail = 10 程度になる。

### 5.2.5 Python 実装

```python
def hill_alpha(values, pct=90):
    x = values[values > 0]
    xmin = np.percentile(x, pct)
    tail = x[x >= xmin]
    if tail.size < 2:
        return np.nan
    log_ratios = np.log(tail / xmin)
    return 1.0 / log_ratios.mean()
```

## 5.3 Bootstrap（信頼区間の構成）

### 5.3.1 問題設定

ρ_S を 100 trial で推定したが、その推定値の信頼区間が欲しい。

### 5.3.2 古典的アプローチ（と限界）

「ρ̂ の標本分布を正規近似 → 標準誤差で CI を構成」。これが Fisher-z の発想。

問題：標本分布が正規でない場合（重尾、歪み）、近似が破綻する。

### 5.3.3 Bootstrap の発想

「標本分布が分からなければ、データから resample して経験的に作る」。

```python
def bootstrap_ci(data, statistic, n_resamples=10000, alpha=0.05):
    boot_stats = []
    for _ in range(n_resamples):
        sample = np.random.choice(data, size=len(data), replace=True)
        boot_stats.append(statistic(sample))
    boot_stats = np.array(boot_stats)
    lo = np.percentile(boot_stats, 100 * alpha / 2)
    hi = np.percentile(boot_stats, 100 * (1 - alpha / 2))
    return lo, hi
```

### 5.3.4 Percentile vs BCa

- **Percentile bootstrap**：上下 2.5 percentile を取る。実装簡単。
- **BCa（bias-corrected accelerated）**：分布のスキュー（偏り）を補正。理論的に望ましいが実装が重い。

YH006_1 は percentile を採用（10,000 resamples）。

### 5.3.5 Bootstrap の限界

- データが少なすぎる場合（n < 30 程度）は CI の精度が落ちる
- 強く相関している観測同士があると、独立性の仮定が破れる
- 「真の分布」がデータに表れていない場合（裾の極端な値が観察されていない）、CI は楽観的になる

## 5.4 Mann-Whitney U test

### 5.4.1 問題設定

「C3 と C3_A1 で ρ_S の分布が違うか」を検定したい。各 100 trial、正規性は仮定したくない。

### 5.4.2 t 検定ではダメな理由

t 検定は両群が正規分布に従い、分散が等しいことを仮定する。trial-level の Spearman は重尾の影響を受けるので、正規分布になる保証がない。

### 5.4.3 Mann-Whitney U の発想

「値ではなく順位を使う」というのは Spearman と同じ発想。

```
1. 全データをまとめて順位をつける
2. グループ A の順位の合計を R_A とする
3. U_A = R_A - n_A(n_A+1)/2 を計算
4. U_A の分布は H0 のもと（両群が同じ分布）で既知 → 棄却域を決める
```

Wilcoxon rank-sum test と本質的に同じ。

### 5.4.4 何を検定しているか

正確には「両群が同じ確率分布から来ている」という H0 を検定する。「平均が等しい」ではなく「分布全体が等しい」。分布の形が違っていても棄却される。

## 5.5 Quantile regression

### 5.5.1 通常の回帰の限界

OLS（最小二乗）は条件付き平均 E[Y|X] を推定する。分布の中央が動かなくても、上下の散らばり（funnel）が変わっていれば、それは見えない。

### 5.5.2 Quantile regression の発想

τ-分位点を回帰する。例えば τ=0.9 なら「90 percentile の Y が X の関数としてどう動くか」。

```
min_β Σ_i ρ_τ(y_i - x_i^T β)
ρ_τ(u) = u · (τ - 1{u < 0})
```

ρ_τ は τ-pinball loss。τ=0.5 なら絶対値（中央値回帰）。

### 5.5.3 Funnel 構造を測る

slope_{0.9} − slope_{0.1} が「上下の散らばりが X とともにどう広がるか」を直接定量化する。これが funnel 構造の指標。

## 5.6 Bin variance

### 5.6.1 発想

「h を log-equal なビンに切り、各ビン内で Var(log|ΔG|) を測る」。

なぜ log-equal か：h の分布が広い範囲をカバーするので、線形ビンだと低 h 側に集中する。log-equal なら各ビンの観測数が比較的均等。

### 5.6.2 何を測っているか

各 h で「条件付き散らばりがどれだけ大きいか」。これが「funnel の開き度」。

Spearman 相関は「中央傾向」を見るが、bin variance は「散らばりの h 依存性」を見る。Funnel が壊れているとは「散らばりの h 依存性が消えている」こと。これを直接測れる。

## 5.7 多重比較補正

### 5.7.1 問題

主指標 + ablation 2つ + 直接指標で4系統の検定を同時にやる。それぞれ α=0.05 で見ると、少なくとも1つで偽陽性が出る確率は約 1 − 0.95^4 ≈ 0.19。

### 5.7.2 Bonferroni 補正

各検定を α/k で見る。k=4 なら α=0.0125。最も保守的。

### 5.7.3 Holm 補正

p 値を昇順に並べて、k 番目の p に α/(k-順位+1) を適用。Bonferroni より検出力が高いが、安全。

YH006_1 は Bonferroni を採用（簡単で安全側）。

## 5.8 量子化と info loss

quantize(Δp, C) で5値に潰すと、価格変化の連続情報が失われる。これはなぜ許容できるか：

- SG の戦略は「過去の量子化された動き」だけを見るので、戦略のレベルでは何も失っていない
- 量子化前の Δp も価格 p の更新に直接使われているので、量子化は「履歴を作るときだけ」の操作
- 5値というのは「強い上昇 / 上昇 / 横ばい / 下落 / 強い下落」という人間の直感的分類に対応

C の値は「±1 と ±2 の境界」。C=3 なら |Δp| > 3 のとき強い動きとカテゴライズ。

# 第6部 Stylized facts の完全リスト

## 6.1 SF1〜SF6 定義

| SF | 名前 | 定義 | 典型値 |
|---|---|---|---|
| SF1 | Absence of return autocorrelations | corr(r_t, r_{t+τ}) ≈ 0 for τ ≥ 1 | <0.05 |
| SF2 | Heavy tails | P(\|r\| > x) ∝ x^{-α}（CCDF が冪乗則） | α ≈ 3（inverse cubic law、分散は有限・kurtosis は無限） |
| SF3 | Aggregational Gaussianity | window 集計で kurtosis が減る | kurt → 0 |
| SF4 | Intermittency | volatility のクラスタリング | corr(|r_t|, |r_{t+τ}|) ≈ 0.3 at τ=1 |
| SF5 | Volatility clustering ACF | corr(|r_t|, |r_{t+τ}|) > 0 for long τ | persistence to τ ~ 100 |
| SF6 | Leverage effect | corr(r_t, |r_{t+τ}|²) < 0 for τ ≥ 0 | ≈ −0.3 |

その他の有名な SF（SF1〜SF6 に含まれないもの）：
- Gain-loss asymmetry：分布の左右非対称
- Volume-volatility correlation：出来高とボラの相関
- Slow decay of |r| ACF：絶対値の自己相関が冪乗的減衰

## 6.2 各 SF の測定法（Python）

```python
import numpy as np

def log_returns(prices):
    p = np.asarray(prices, dtype=float)
    p = np.where(p > 0, p, np.nan)
    return np.diff(np.log(p))

def return_acf(returns, max_lag=50):
    r = returns[~np.isnan(returns)]
    r = r - r.mean()
    var = (r ** 2).mean()
    if var == 0:
        return np.zeros(max_lag)
    return np.array([(r[:-lag] * r[lag:]).mean() / var for lag in range(1, max_lag + 1)])

def volatility_acf(returns, max_lag=500):
    return return_acf(np.abs(returns), max_lag)

def hill_alpha(values, pct=90):
    x = values[values > 0]
    xmin = np.percentile(x, pct)
    tail = x[x >= xmin]
    if tail.size < 2: return np.nan
    return 1.0 / np.log(tail / xmin).mean()

def kurtosis_window(returns, window):
    r = returns[~np.isnan(returns)]
    if window > 1:
        n = (len(r) // window) * window
        r = r[:n].reshape(-1, window).sum(axis=1)
    z = (r - r.mean()) / r.std()
    return (z ** 4).mean() - 3.0
```

## 6.3 SG が再現するもの / しないもの

**再現するもの**（Katahira 2019 論文の図 1〜10）：
- SF1: r ACF ≈ 0
- SF2: heavy tails（α ≈ 3〜4 を内生的に生成）
- SF3: aggregational Gaussianity
- SF4, SF5: volatility clustering
- 富分布の Pareto テール

**再現しないもの**（論文 1 の図 11〜13）：
- 時間スケール非対称性
- SF6: レバレッジ効果
- gain-loss 非対称性

理由は Q9 の通り。戦略テーブルが price-direction 対称、q が w しか見ない。

# 第7部 想定問答100

## 7.1 基礎概念（30問）

1. **Q: ABM とは何ですか。**
   A: Agent-Based Model。多数の異質エージェントが局所相互作用するシステムをシミュレーションするモデル化手法。マクロ現象をミクロから積み上げて再現する。

2. **Q: なぜ均衡理論では金融市場が説明できないのですか。**
   A: 均衡理論は代表的エージェントを仮定するので、参加者の異質性、局所相互作用、適応、創発の4要素が消える。実証される厚い裾やボラティリティが固まる現象はミクロの相互作用の積み重ねからしか出ない。

3. **Q: stylized facts とは何ですか。3つ挙げてください。**
   A: 複数の市場・期間・資産で繰り返し観察される統計的指紋。例：(1) 厚い裾、(2) ボラティリティが固まる現象、(3) 集約的正規化。

4. **Q: 厚い裾とは具体的に何ですか。**
   A: 高頻度の収益分布が正規分布より極端な値を頻繁に出す性質。CCDF が冪乗則 P(|r| > x) ∝ x^{-α} に従い、金融市場では α ≈ 3 が観測されることが多い（"inverse cubic law"）。α ≤ 2 なら分散自体が無限になり、Mandelbrot の stable Paretian 仮説の領域。α ≈ 3 は「分散は有限だが kurtosis は無限」の領域。

5. **Q: ボラティリティが固まるとは何ですか。**
   A: 収益自体の自己相関はゼロだが、収益の絶対値や2乗の自己相関は数日〜数週間続く。大きな動きの後は大きな動きが続く。

6. **Q: 集約的正規化とは何ですか。**
   A: 時間スケールを長くする（日次→月次→年次）と分布が正規分布に近づく性質。短時間スケールでは裾が厚い。

7. **Q: レバレッジ効果とは何ですか。**
   A: 価格が下がった後にボラティリティが上がる相関。相関係数で −0.3 程度。

8. **Q: Speculation Game とは何ですか。**
   A: Katahira-Chen 2019 の ABM。戦略テーブル、口座残高による発注量、需給集計型の価格更新の3層構造を持つ。

9. **Q: SG の3層構造を説明してください。**
   A: 第1層が戦略の層（過去 M 期間の量子化履歴から {買い/売り/持ち越し} を返す表）、第2層が口座残高の層（q = ⌊w/B⌋ で発注量を決める）、第3層が価格形成の層（需給集計で Δp を作る）。

10. **Q: Minority Game との違いは何ですか。**
    A: SG は富が発注量に直接効く経路（q = ⌊w/B⌋）を持つ。MG は仮想点数のみで発注量に効かない。これが決定的な違い。

11. **Q: SG の戦略はどう更新されますか。**
    A: virtual reinforcement。使っていない戦略も「もし使っていたら」という仮想ポジションを保持し、close 相当で ΔG_v を G に加える。active の評価が他より低くなったら最大値の戦略に乗り換える。

12. **Q: 破産退場の規則は何ですか。**
    A: 口座残高 w が 1 単元のコスト B を下回ったら、そのエージェントを取り除き、新規エージェントに置き換える。戦略・残高・状態を全部リセット。

13. **Q: 破産退場が止まると何が起きますか。**
    A: SG の口座残高分布の冪乗則的定常状態が壊れる。退場と再生のフローが分布を作るエンジンになっている。

14. **Q: 需給集計型とは何ですか。**
    A: Δp = (1/N)Σ a_i q_i で価格を機械的に更新するモデル。100% 約定。スプレッド・指値・キャンセル・部分約定は概念として存在しない。

15. **Q: LOB（Limit Order Book）とは何ですか。**
    A: 現実の取引所の仕組み。指値で板に注文を置き、価格優先・時間優先のマッチングで売買が対面したときだけ約定する。約定の不確実性、スプレッド、流動性の偏りがある。

16. **Q: PAMS とは何ですか。**
    A: Hirano-Izumi 2023 の Python の金融 ABM フレームワーク。Limit Order Book を実装している。YH006 は SG の決定ルールを PAMS 上に移植した。

17. **Q: FCNAgent とは何ですか。**
    A: Fundamental-Chartist-Noise の3要素を組み合わせた基本トレーダー。fundamental（理論価値に従う）、chartist（過去の trend に従う）、noise（ランダム）を重みづけて行動する。

18. **Q: なぜ MMFCNAgent を使ったのですか。**
    A: PAMS 0.2.2 の FCNAgent.submit_orders_by_market が order_volume を 1 にハードコードしていたので、order_volume=30 で出させる回避策。これが YH006 の「外部流動性条件」になっている。

19. **Q: c_ticks とは何ですか。**
    A: LOB の tick scale（最小値刻み）を、aggregate 世界の典型的な |Δp| に合わせるための較正値。C1（FCN のみの LOB ベースライン）の median|Δmid| × 3 で決定。YH006 では c_ticks ≈ 28.0 tick。

20. **Q: なぜ N=100 で揃えたのですか。**
    A: 旧来比較では N=1000（aggregate）vs N=100（LOB）で「N の効果」と「世界の効果」が混ざっていた。N=100 で揃えると、Hill 指数のような N 依存指標も純粋に世界効果だけ取り出せる。

21. **Q: substitute とは何ですか。**
    A: 破産退場と新規参入の操作。w < B のエージェントを取り除いて新規に置き換える。SG の定常分布を維持する核心の仕組み。

22. **Q: q = ⌊w/B⌋ の意味を説明してください。**
    A: エージェントの発注量。口座残高 w を1単元コスト B で割った整数。儲かれば次に大きく張れて、損すれば小さくしか張れない、というフィードバック。

23. **Q: 認知価格 P と実価格 p の違いは何ですか。**
    A: p は連続値の実価格、Δp で更新される。P は量子化された h の累積（整数）、戦略の損益計算に使う「価格信号」。エージェントから見える価格は P の方。

24. **Q: なぜ価格を量子化するのですか。**
    A: 戦略テーブルが「離散的な履歴 index」を入力に取るので、連続な Δp を5値に潰す必要がある。「強い上昇/上昇/横ばい/下落/強い下落」という人間の直感的分類に対応。

25. **Q: virtual position とは何ですか。**
    A: active でない戦略について保持する「もし使っていたら」の仮想ポジション。仮想 open / 仮想 close で ΔG_v を計算し、戦略の評価 G に加える。

26. **Q: ablation とは何ですか。**
    A: モデルの特定の機構経路を切ったり固定したりして、結果がどう変わるかを見る実験手法。仮説 A が想定する経路を切れば、結果が予測通りに変わるはず、というロジックで仮説検証する。

27. **Q: 仮説 A はどんな仮説ですか。**
    A: 「LOB 摩擦が初期富分布の persistence を伸ばし、q = ⌊w/B⌋ 経路で注文サイズの不均一性が固まる」が funnel 構造の interaction を作る、という機構仮説。

28. **Q: 仮説 B はどんな仮説ですか。**
    A: 上位 wealth agent が選択的に長/短 horizon の round-trip を取る、という戦略選択仮説。q 経路を切っても効くはず。

29. **Q: 仮説 C はどんな仮説ですか。**
    A: 大 q order ほど LOB で fill されにくく、wealth 依存的に約定パターンが変わる、という fill asymmetry 仮説。

30. **Q: 三つの仮説をどう区別しますか。**
    A: A1 ablation（q を全員一律固定）と A3 ablation（寿命に上限）の効き方の組合せで。A1+A3 両方効く → A、A3 だけ効く → C、両方効かない → B。

## 7.2 実装（20問）

31. **Q: SG をゼロから書くなら、最初のステップは何ですか。**
    A: 初期化。N, M, S を受け取って、戦略テーブル（N×S×K の3次元配列、K=5^M）、口座残高、active 戦略 index、初期履歴を生成する。

32. **Q: K = 5^M はなぜ 5 の M 乗ですか。**
    A: 各履歴位置が5値量子化されているので、M 期間の組合せは 5^M 通り。

33. **Q: shift_in の式を書いてください。**
    A: `return ((mu * 5) + new_value) % K`。5進数として末尾に new_value を入れる。

34. **Q: 量子化関数を書いてください。**
    A: dp > C なら 2、dp > 0 なら 1、dp < -C なら -2、dp < 0 なら -1、それ以外 0。

35. **Q: 各 step の処理順序を9段階で言ってください。**
    A: (1) μ snapshot、(2) 推奨アクション、(3) effective action 表、(4) 需給集計と価格更新、(5) 量子化と履歴更新、(6) close 実行（記録 → 状態クリア）、(7) open 実行、(8) virtual 更新、(9) 戦略レビュー + 破産判定。

36. **Q: close 時の ΔG の式は？**
    A: `dG = entry_action * (P_now - entry_price)`。買い建てなら P 上昇で +、売り建てなら P 下降で +。

37. **Q: close 時の w 更新は？**
    A: `w += dG * entry_quantity`。1単元あたりの ΔG が entry_quantity 倍されて残高に反映される。

38. **Q: なぜ close と open を別ループにしますか。**
    A: 同じ step 内で、ある i が close した結果（w 更新）が、別の j の open の quantity に影響しないように。同期更新を保つため。

39. **Q: 戦略レビューでタイブレークはどう扱いますか。**
    A: argmax の集合 best にすでに active_idx が含まれていれば続行。含まれていなければ best からランダムに新しい index を選ぶ。

40. **Q: 破産後の新規エージェントの初期化を書いてください。**
    A: strategies = rng.choice([-1,0,1], (S,K))、w = B + rng.integers(0,100)、active = rng.integers(0,S)、position 等は 0。G もゼロクリア。

41. **Q: なぜ vectorized 版が必要ですか。**
    A: per-agent ループは N=100, T=50000 で数十秒。ベクトル化版なら数秒。100 試行のアンサンブルでは差が大きい。

42. **Q: bit-parity を保つにはどうしますか。**
    A: RNG 消費順序を厳守する。同じ seed で、同じ順序で乱数を消費すれば、結果は1ビットも違わない。

43. **Q: round_trips のデータ構造は？**
    A: agent_idx, open_t, close_t, entry_action, entry_quantity, delta_G を並列の numpy 配列として保持。

44. **Q: 横軸 horizon、縦軸 ΔG の散布図を作るには？**
    A: horizon = close_t - open_t、dG = delta_G として、hexbin プロット。

45. **Q: corr(|ΔG|, h) の計算を書いてください。**
    A: `np.corrcoef(np.abs(dG), horizon)[0, 1]` で Pearson。Spearman は scipy.stats.spearmanr。

46. **Q: LOB に SG を移植する最大の難所は何ですか。**
    A: SG の認知価格 P と LOB の mid-price がスケールが違うこと。c_ticks 較正で対応する。また zero-fill open 率が高い（注文が約定しない）ことへの workaround も必要。

47. **Q: PAMS の Agent サブクラスをどう作りますか。**
    A: `pams.agents.Agent` を継承して、`submit_orders` メソッドをオーバーライド。SG decision rule（rec を引いて、open/close/hold を判定し、quantity = w//B で注文を出す）を実装。

48. **Q: PAMS と SG で共有する状態は何ですか。**
    A: 認知履歴 μ。全 SG エージェントで共有される cognitive history を別オブジェクトとして持ち、step 末に1回更新する。

49. **Q: テストは何を書きますか。**
    A: (a) parity test: seed 固定で bit 一致、(b) round-trip invariants: open_t < close_t, entry_action ∈ {±1}, etc、(c) wealth conservation: sg_wealth ≥ 0 かつ有界、(d) aggregate parity: uniform モードで YH005 と bit 一致。

50. **Q: 実行時間の見積もりは？**
    A: Apple Silicon シングルスレッドで、C0u/C0p 各 30 秒、C1 56 秒、C2/C3 各 15 秒、計 2.4 分（Phase 1 単 trial）。Phase 2 で 7 条件 × 100 trial だと並列化前で約 700 倍、並列化（8 core）で約 90 倍 = 数時間〜数十時間。

## 7.3 統計手法（20問）

51. **Q: なぜ Spearman を主指標にしたのですか。**
    A: Pearson は重尾分布で分散が不安定になり、外れ値1個に引きずられて偽相関を作る可能性がある。Spearman は順位ベースなので単調変換不変、外れ値の影響を受けにくい。

52. **Q: Pearson と Spearman を両方計算するのはなぜですか。**
    A: Pearson 単独だと artifact を拾う可能性、Spearman 単独だと検出感度が下がる。両方で符号と桁が一致すれば「実際に単調関係がある」ことの強い証拠。

53. **Q: Kendall を補助に使う理由は？**
    A: Spearman より外れ値に頑健（ペアの順序の一致だけを見るので）。3つ並走させて、すべて同方向なら本物、食い違ったらどれかが artifact。

54. **Q: bootstrap で何を計算しますか。**
    A: 100 trial の Spearman ρ から、resample = 10,000 で経験分布を作り、percentile で 95% CI を構成。

55. **Q: なぜ Fisher-z 変換を使わないのですか。**
    A: Fisher-z（z = arctanh(ρ)）は ρ の標本分布を近似正規にする変換だが、重尾分布のもとで近似が破綻する。Bootstrap は分布を仮定しないので頑健。

56. **Q: percentile bootstrap と BCa の違いは？**
    A: Percentile は上下 2.5% を取るだけ、実装簡単。BCa は分布のスキュー（偏り）と加速度を補正、理論的に望ましいが実装が重い。YH006_1 は percentile を採用。

57. **Q: Mann-Whitney U で何を検定しますか。**
    A: 「両群が同じ確率分布から来ている」を H0 にする。平均だけでなく分布全体を比べる。順位ベースなので正規性は不要。

58. **Q: 多重比較補正はなぜ必要ですか。**
    A: 主指標 + ablation 2つ + 直接指標で4系統を同時に検定すると、各 α=0.05 だと少なくとも1つ偽陽性が出る確率が約 19%。

59. **Q: Bonferroni と Holm の違いは？**
    A: Bonferroni は全検定を α/k で見る（最も保守的）。Holm は p 値を昇順に並べて、k 番目に α/(k-順位+1) を適用（検出力高い）。YH006_1 は Bonferroni 採用。

60. **Q: Hill 推定量とは何ですか。**
    A: 重尾分布のテール指数 α の最尤推定値。サンプル降順で上位 k 個を取り、log(X_{(i)}/X_{(k+1)}) の平均の逆数。α が小さいほど裾が重く、α ≤ k で k 次モーメントが発散する（α ≤ 1 で平均発散、α ≤ 2 で分散発散、α ≤ 4 で kurtosis 発散）。

61. **Q: Hill の k はどう決めますか。**
    A: 小さすぎると分散大、大きすぎるとバイアス。典型的に k = √n か、xmin = 90 percentile を境界。YH006 では p90。

62. **Q: quantile regression は何を測りますか。**
    A: 条件付き τ-分位点が説明変数の関数としてどう動くか。slope_{0.9} − slope_{0.1} で「上下の散らばりが X とともにどう広がるか」を直接定量化。

63. **Q: bin variance はなぜ funnel 構造の直接指標になるのですか。**
    A: 各 h ビン内で Var(log|ΔG|) を測ることで、h の関数として「散らばり」を見る。Spearman は中央傾向しか測らないが、bin variance は分散の h 依存性を見る。

64. **Q: なぜ log-equal なビンですか。**
    A: h の分布が広い範囲にわたるので、線形ビンだと低 h 側に観測が集中する。log-equal なら各ビンの観測数が比較的均等。

65. **Q: paired seed と independent seed の違いは？**
    A: Paired は条件間で同じ seed を使う（比較の精度が高い）。Independent は条件ごとに違う seed（実装単純、頑健性優先）。YH006_1 は independent。

66. **Q: 100 trial で十分ですか。**
    A: Spearman ρ の検出力は条件次第。Cohen's d で効果量が中程度なら 100 trial で 80% の検出力。Phase 1 観察値 −0.27 は中〜大の効果量なので、100 で十分と判断。

67. **Q: timescale sensitivity とは何ですか。**
    A: 各 trial の round-trip を時刻順に前半/後半に分割し、それぞれで Spearman を計算。後半で interaction が縮小していれば「初期分布の流れ」が観察される。完全に流れていれば消える。

68. **Q: wealth persistence の半減期とは？**
    A: corr(w_init, w(t)) を時刻 t = T/10, 2T/10, ..., T で計測し、相関が 0.5 になる時刻。LOB > aggregate なら「LOB は流れが遅い」の定量証拠。

69. **Q: 検定の有意水準を α = 0.05/4 = 0.0125 にする根拠は？**
    A: Bonferroni 補正。4系統の検定で family-wise error rate を 0.05 以下に保つ。

70. **Q: 統計的有意性と効果サイズの違いは？**
    A: 統計的有意性は「偶然では起きにくい」を保証するだけ。効果サイズは「実際にどれだけ大きい効果か」を測る。N が大きければ小さい効果も有意になるので、両方報告する必要がある。

## 7.4 YH006 / YH006_1（20問）

71. **Q: YH006 で何を比較しましたか。**
    A: 世界軸（aggregate vs LOB）× 初期残高軸（uniform vs Pareto α=1.5）の 2×2、N=100 統一。加えて FCN-only ベースラインの C1。

72. **Q: YH006 の主結果を一言で言うと？**
    A: corr(|ΔG|, h) の交互作用 −0.27。aggregate では Pareto 初期化に頑健、LOB では Pareto 初期化で大きく劣化。

73. **Q: なぜ −0.27 が「明瞭な交互作用」と言えるのですか。**
    A: aggregate での Pareto 効果が −0.006 とほぼゼロ、LOB での Pareto 効果が −0.28。差分の差分が −0.27 で、これは「世界によって富軸の効果の大きさが違う」という形の交互作用。

74. **Q: Phase 1 の結果を定量的に主張できないのはなぜですか。**
    A: 単 trial だから。bootstrap CI が取れていない。Phase 2 で 100 trial × 7 条件のアンサンブルで初めて定量的有意性が言える。

75. **Q: Stage 5.8 で何が分かりましたか。**
    A: 板型シミュレーションでは 500 万 agent-step の間に強制退場イベントがほぼゼロ（hazard rate の 95% 上限が 5.9e-7）。需給集計型では退場が定常的。

76. **Q: 破産退場ゼロは何を意味しますか。**
    A: LOB 摩擦下では、SG の口座残高ダイナミクスの自己組織化（破産と再生による定常状態の維持）が物理的に止まっている可能性。

77. **Q: YH006_1 の唯一の問いを言ってください。**
    A: LOB 環境で観察された corr(|ΔG|, h) の交互作用 −0.27 は、SG の q = ⌊w/B⌋ 経路に causal に起因するか。

78. **Q: A1 ablation で何を変えますか。**
    A: 通常 q_i = ⌊w_i/B⌋ を、全エージェントで q = q_const（固定値）に置き換える。q_const は pilot で median_{t,i}(q_i(t)) の median を取る。

79. **Q: なぜ q_const を中央値ベースで決めるのですか。**
    A: Pareto 重尾下で平均は不安定。中央値は典型エージェントの活動量を保存する。

80. **Q: A3 ablation で何を変えますか。**
    A: 各エージェントに「τ_max ステップ滞在したら強制 substitute」を加える。これで lifetime persistence の効きを切る。

81. **Q: KPI L1 から L4 の階層を説明してください。**
    A: L1: 指標横断で signature の robustness 確認、L2: A1 で interaction の 50% 以上縮小、L3: A3 でも部分縮小 + 中間予測整合、L4: L1+L2+L3 すべて + funnel 直接指標一致 + timescale で long-run でも消えない。

82. **Q: L2 で proposal の核として機能するのはなぜですか。**
    A: A1 で interaction が消える = 「q 経路を切れば現象が消える」が証明される = 機構が一意に identify される。これがあれば「SG の核心機構が LOB でも生きている」を定量的に主張できる。

83. **Q: Plan B のシナリオ α/β/γ/δ/ε を言ってください。**
    A: α: F1 本物、β: Pearson 重尾 artifact、γ: 分散構造のみ変質、δ: LOB は流れが遅いだけ、ε: A1 で消えず仮説 B か未知機構。

84. **Q: 最も恐れているシナリオはどれですか。**
    A: β（Pearson が拾ったのが artifact）。これだと主結果が崩れる。だから Spearman/Kendall/bin variance の3つで並走確認している。

85. **Q: なぜ S1（既存データの再分析）を最優先にするのですか。**
    A: 新規 simulation 不要で 2-3 日で完了、結果次第で S2 以降の意味が大きく変わるから。S1 で β/γ なら方針転換。

86. **Q: ファネル構造とは何ですか。**
    A: 横軸 horizon、縦軸 ΔG で散布図を取ると、horizon が大きくなるにつれて ΔG の散布が広がる構造（漏斗状）。富分布が定常状態に達した SG の特徴。

87. **Q: なぜ funnel が機構の指標になるのですか。**
    A: SG の口座残高ダイナミクスが正常に働いていれば、富のスケールが horizon と比例的に増える。これが ΔG のスケールに反映され、funnel として現れる。funnel が壊れていれば、機構が壊れている。

88. **Q: 中間予測（chain の link 検証）で何を見ますか。**
    A: agent lifetime 分布、wealth persistence corr(w_init, w_final)、forced retirement rate、order size persistence、round-trip 開始時 wealth と horizon の joint。

89. **Q: 仮説 A の chain がどこで切れているかをどう特定しますか。**
    A: chain の各 link に対応する中間予測を全部測る。例えば lifetime が C3 で C2 より長くなければ「LOB で生存しやすい」が嘘、つまり chain の最初の link が崩れる。

90. **Q: pilot を最初にやる理由は何ですか。**
    A: q_const と τ_max のキャリブレーションが事前に必要。本実験前に 5 trial で確認しないと、ablation が機能しているかどうか分からないリスクがある。

## 7.5 研究計画（10問）

91. **Q: あなたの研究の中心的な問いを一文で言ってください。**
    A: 「金融 ABM が stylized facts を再現するだけでなく、その背後のミクロ機構をどこまで識別できるか」です。

92. **Q: なぜ Speculation Game を選んだのですか。**
    A: SG は「認知的意思決定」と「富の動的分布」が結合した最小モデル。3層構造が単純で、機構の切り分け実験がやりやすい。また Chen 先生たちのオリジナルなので、設計意図への理解が深いラボで議論できる。

93. **Q: 修士で何を達成しますか。**
    A: YH006 で見えた予備結果を厳密化するために、注文サイズ、約定率、価格形成、初期資産分布への感度分析と ablation で、どの機構が効いているかを切り分けます。

94. **Q: 修士後はどう発展させますか。**
    A: SG が再現できていない時間スケール非対称・レバレッジ効果・利得損失非対称に取り組む。これにはプロスペクト理論などの状況依存 q を入れる必要があり、修士後の課題。

95. **Q: 競合する研究は何ですか。**
    A: Lux-Marchesi、Cont-Bouchaud、Franke-Westerhoff などの古典 ABM。これらは富のフィードバックを持たず、SG の特徴と差別化される。また Hirano-Izumi の PAMS は流動性供給側のモデリングが主で、SG とは補完関係。

96. **Q: 失敗するとしたら何が原因ですか。**
    A: (1) Phase 2 で 100 trial 取っても交互作用が有意にならない、(2) A1 ablation が機能しない（q を固定すると SG が壊れる）、(3) LOB の workaround（MMFCNAgent）が結果に支配的に影響。これらはあらかじめ Plan B で対応を用意している。

97. **Q: 何が一番の貢献ですか。**
    A: SG の核心機構である「富のフィードバック経路」が、需給集計型から LOB に世界を変えたときに変質するか否かを、ablation を通じて初めて切り分ける。これは機構の identifiability に関する方法論的貢献も持つ。

98. **Q: なぜ陳先生の研究室か。**
    A: 陳先生の研究室には金融 ABM と Speculation Game 系の研究蓄積があり、自分が立てている問いの土台と直接重なるからです。SG をベースに LOB のような微視構造を持つ市場へ拡張するのは自分が持ち込む部分で、SG への理解が深いラボでないと議論が噛み合わないと考えています。

99. **Q: ABM 全般への展望は？**
    A: stylized facts での再現性争いから、機構の identifiability に重心が移る時期だと考えています。観察データだけでは複数機構を分離できないので、市場環境を変えたときの応答差を使った切り分けが必要。

100. **Q: あなたの研究は誰の役に立ちますか。**
     A: 直接の受益者は ABM の研究者コミュニティ。間接的には、市場制度の評価に ABM を使いたい規制機関や取引所。ただし、修士の段階では基礎研究としての方法論的貢献に留めます。

## 8.1 手計算問題

**問題 1**：M=2 のとき K = ?

**問題 2**：M=3 で履歴 (h_{-3}, h_{-2}, h_{-1}) = (+1, -2, 0) のとき、5進数表現を計算して μ を求めよ。
（ヒント：h+2 で {0, 1, 2, 3, 4} に変換）

**問題 3**：B=9、初期 w=50 のエージェントが open するときの q は？ その後 ΔG=+3 で close したときの新しい w は？

**問題 4**：N=100 で全員が同じ a=+1, q=10 で発注するとき、Δp は？

**問題 5**：Δp = 4.5、C = 3.0 のとき、h = ?

**問題 6**：5 試行の Spearman ρ が [0.61, 0.58, 0.65, 0.55, 0.63] のとき、平均と標準誤差を計算せよ。

**問題 7**：11 個のサンプル [1, 2, 3, 5, 7, 10, 15, 25, 50, 100, 200] の Hill α を、numpy.percentile（default linear）で xmin = p90 を計算した上で求めよ。（n=11 だと p90 はサンプル値そのものに一致するので確認しやすい）

**問題 8**：Bonferroni 補正で、k=4 系統の検定をするときの個別 α は？

**問題 9**：bootstrap で 10,000 回 resample したとき、95% CI の上下限は経験分布のどの percentile？

**問題 10**：YH006 Phase 1 で corr(|ΔG|, h) が C0u: 0.353, C0p: 0.347, C2: 0.61, C3: 0.33 のとき、交互作用効果を計算せよ。

### 解答

1. K = 5² = 25
2. (h+2) = (3, 0, 2) → μ = 3·25 + 0·5 + 2 = 77
3. q = ⌊50/9⌋ = 5。close 後 w = 50 + 3·5 = 65
4. Δp = (1/100)·(100·10) = 10
5. dp=4.5 > C=3 なので h = 2
6. 平均 = 3.02/5 = 0.604。偏差 = (0.006, −0.024, 0.046, −0.054, 0.026)、偏差²の和 = 0.000036 + 0.000576 + 0.002116 + 0.002916 + 0.000676 = 0.006320。SD（n−1で割る不偏推定）= sqrt(0.006320/4) = sqrt(0.001580) ≈ 0.0397。SE = SD/√n = 0.0397/√5 ≈ 0.0178。
7. n=11、p90 の position = 0.9·(11−1) = 9、sorted の index 9 の値 = 100。よって xmin = 100。tail = [100, 200]、log(100/100) = 0、log(200/100) = ln 2 ≈ 0.693、mean = 0.347、α = 1/0.347 ≈ 2.88。注: 元の 10 サンプル版だと p90 = 55（線形補間で arr[8]=50 と arr[9]=100 の間 8.1 位置）になり tail = [100] のみ、計算不能になる。
8. α = 0.05/4 = 0.0125
9. 2.5 percentile と 97.5 percentile
10. (C3 − C2) − (C0p − C0u) = (0.33 − 0.61) − (0.347 − 0.353) = −0.28 − (−0.006) = −0.274 ≈ −0.27

## 8.3 概念問題

**問題 16**：「SG が leverage effect を再現できない」のはなぜか。原典の説明と自分の解釈の両方を言え。

### 解答（要点）

16. 原典：戦略テーブルが上昇/下落パターンを対称に扱う。自分の解釈：q = ⌊w/B⌋ で発注量が口座残高だけで決まり、含み損益状態を見ないので、下落特有の反応が組み込まれていない。

17. (a) SF は隠れ状態を見ない（センチメント・情報の偏在は反映されない）、(b) SF は market environment の応答を測らない（呼値刻みを変えたときの反応は SF では分からない）、(c) SF は機構の識別を保証しない（複数機構が同じ SF を作れる）。

18. β シナリオ。「LOB でも SG の機構は頑健に働く」という positive finding として論文化。SG の implementation-invariance を Katahira 2019 の核心主張の強化として書く。

19. q の決定式を、含み損益 P&L_unrealized を引数に持つ関数に拡張：q = f(w, P&L_unrealized, λ)。例えば含み益で q を縮小、含み損で q を拡大。

20. (a) 重尾分布で Pearson が外れ値1個に引きずられた、(b) horizon と |ΔG| の同時分布が二次曲線的で、線形相関が偽の単調性を拾った、(c) seed 依存（単 trial なので偶然の結果）。
