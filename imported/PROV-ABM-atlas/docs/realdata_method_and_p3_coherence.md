# 実データ照合の手法とデータ要件、および P3 との整合

**2026-06-13**。本稿は (1) 案1 を実データで実行する手法とデータ要件（本環境では実行不能なため
仕様として記す）、(2) P1 と P3（LLM-ABM 監査）の整合、を述べる。

## 1. 実データ照合の手法（JPX 2014 tick 改革）

目的: 連続市場の歴史データでは区別できない複数 ABM のうち、実在の改革の前後データでどれの予測が
当たるかでモデルを選ぶ。

手順:
1. **改革前データへの較正**: JPX 2014-01-14 の tick 引き下げ前（例 2013-01〜2014-01）の対象銘柄
   データに、価格を読む ABM と注文流を読む ABM をそれぞれ較正する。両者が改革前データに同程度
   当てはまることを確認する（当てはまりに差があれば、その時点で当てはまりの良い方を選べる）。
2. **改革後の予測**: 各 ABM が、tick 引き下げ後の市場の量を予測する。両者の予測が異なる量を用いる。
3. **実データとの照合**: 改革後の実データ（2014-01〜2015-01）で、どちらの予測が当たるかを測る。
4. **選択**: 当たった方を、その市場を記述する ABM として採る。

## 2. 測る量（PRISM の非検出を避ける）

我々の予備的検討では、tick 改革の効果を日次の return-distribution 量（GARCH persistence・尖度・
歪度・自己相関）で測ると、有効なサンプル（40 treatment + 20 control 銘柄、12ヶ月）でも全量の 95%
信頼区間がゼロを跨ぎ、有意な効果を検出できなかった。tick 改革の効果は spread・depth・price impact
という microstructure 量に現れ（Aquilina, Budish & O'Neill 2022、Comerton-Forde, Gregoire & Zhong
2019 が実測）、日次 return 分布にはほとんど現れない。よって照合に用いる量は **microstructure 量を
intraday 周波数で**測る必要がある:

- 実効スプレッド、quoted spread
- 板厚（depth）、price impact（Kyle λ）
- 約定の到着率・サイズ分布、odd-lot 比率

## 3. データ要件

- **必要**: JPX の intraday データ（板・約定）。J-Quants API（intraday）または取引所データ。対象は
  TOPIX 100（treatment）+ 非 TOPIX 100（control）、2013-01〜2015-01。
- **不可**: yfinance 等の日次データのみでは、§2 の通り tick 効果が return 分布に現れず照合できない。
- **本環境の制約**: 本作業環境はネット接続なし・yfinance/J-Quants 未導入のため、データ取得・照合は
  実行できない。手法と要件のみを上記に記す。J-Quants アクセスのある環境で実行する。

## 4. P3（LLM-ABM 監査）との整合

P3 は公表された LLM-ABM の主張を監査する（再現 → 分散測定 → 介入統制）。案1 で示した「介入が、
歴史データでは決まらないモデルの違いを区別する」は、P3 の第3段（介入統制）に具体的手段を与える。

- LLM-ABM が「エージェントは X を見て取引する」と主張するとき、X とデータ上区別できない別チャネル
  Y があれば、再現（観測適合）だけでは「X を見ている」か「Y を見て X に見えるだけ」かを決められない。
- 案1 の介入（チャネルを分離する政策）は、X と Y を分離してモデルの出力がどちらに従うかを区別する。
  P3 はこれを LLM-ABM に適用する: 主張された入力チャネルを分離する介入を加え、出力が追随するかを見る。
- 前提: 被監査モデルが観測チャネルを宣言する（Model Contract C1）。宣言がなければ介入面が定義できず、
  再現（C0）以上の監査ができない。案1 はチャネル宣言が監査に必須であることを示す。
- 対象が金融 LLM-ABM（TwinMarket 級）なら、案1 の介入（batch/tick によるチャネル分離）が直接使える。
- 整合上の限界: 案1 の2要件（観測等価かつ介入応答差）は P3 にも要る。監査の介入は、主張機構と帰無
  機構が異なる応答を示し、かつ両者が再現を満たすものでなければならない。さらに、§2 と同様、介入が
  効果を与える量を測る必要がある。
- **honest な留保**: 単一資産市場での案1 の識別精度は、価格を読む機構が batch 下でどう振る舞うかに
  依存する。価格を読む単純なモメンタム取引者は、batch 下でバッチ内の価格が動かない間は取引を減らす
  （これは bug でなく機構の予測）。この取引減少が強い識別（CNN≈0.94）を生む。一方、価格変化イベント
  に適応して取引を続ける版では識別は modest（CNN≈0.63）にとどまる。識別の強さは 0.63〜0.94 の幅を
  持ち、取引者が改革にどう適応するかに依存する。これは案2（学習 vs 固定）と同じ依存である。P3 への
  移転時も、介入が強い識別を与えるのは、被監査モデルが介入に対し主張機構どおりの差を生む場合に限る。
