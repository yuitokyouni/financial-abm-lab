## 0. Why a contract

Gym became the RL standard through the `env.step()` *contract*, not an algorithm.
The Atlas's adoption surface is the **simulator-side** contract: "implement this
interface and your model becomes auditable on the Atlas battery." The contract
decides *auditability*, **not** validity ŌĆö the Atlas does not rank whether a model
is realistic (spec 002 ┬¦1 non-goal). Arena formation begins from this contract, not
from a leaderboard (spec 002 ┬¦3).

## 1. The contract surface (v1)

```python
class Simulator(Protocol):
    name: str
    channels: tuple[str, ...]          # exposed B2 observation channels (the interventionķØó)

    def reset(self, seed: int) -> None: ...        # deterministic init: (config, seed) ŌåÆ bit-identical
    def step(self, action=None) -> None: ...       # advance 1 tick (dt is model-declared)
    def observe(self, channel: str) -> float | NDArray: ...  # post-intervention value; undeclared ŌåÆ error
    def intervene(self, do: Intervention) -> None: ...       # do(channel, scheme, ╬Ė); graded degrade only
    def emit(self) -> Mapping[str, NDArray]: ...   # output series (SF battery + Response input)
    def emit_prov(self) -> dict: ...               # L2 provenance sidecar (seed/config/output digest)


@dataclass(frozen=True)
class Intervention:
    channel: str     # a declared channel
    scheme: str      # one of the four B2 schemes (atlas.intervention.InterventionScheme: a|b|c|d)
    theta: float     # graded strength; ╬Ė = 0 MUST be the identity (no-op)
```

Canonical `abm_models` models expose `run(*, seed) -> dict` only. The adapter
`atlas.contract.ContractModel` maps `run(seed)` ŌåÆ `reset(seed)` + `emit()` and
synthesizes `emit_prov()`. Such models satisfy **C0 (+C2)** but not C1 (no exposed
channel) ŌĆö see ┬¦3.

## 2. Design principles the contract enforces

1. **InterventionķØó = observation channels only (B2 ŌēĀ A).** `intervene` degrades a
   *declared observation channel*; it never touches mechanism coefficients
   (ablation is out of contract). The four schemes live in `atlas.intervention`.
2. **╬Ė = 0 is the identity** (property-tested in `tests/test_intervention.py`).
3. **Determinism is the bedrock.** `reset(seed)` must give bit-identical numeric
   output for the same `(config, seed)`; a non-deterministic model is not even C0.
4. **Exposed channels ŌēĀ declared channels.** `channels` is what the contract
   surface *actually offers now*. A model's *intended* mechanism channels live in
   the frozen semantic registry (`atlas.registry`) as `declared_channels`. The
   Atlas records both so neither is overstated (spec 002 ┬¦4 / ┬¦11). Every canonical
   `run(seed)` model has `channels = ()` until the order-book channel work
   (PROV-ABM-atlas Finding 0002) wires an exposed surface.

## 3. Conformance levels

| Level | Requires | Enables |
|---|---|---|
| **C0** | `reset` / `emit` + determinism | SF battery, replication, variance measurement |
| **C1** | + non-empty *exposed* `channels` / `observe` / `intervene` | intervention-response protocol (the P1 main experiment) |
| **C2** | + `emit_prov` (L2) | declared-vs-observed reconciliation (reach audit) |

`atlas.contract.detect_conformance` returns the highest level a model actually
satisfies. C1 is gated on an *exposed* channel surface, so canonical models report
**C2** (deterministic + prov) but not C1. This is recorded honestly, never faked.

## 4. Submission gate vs semantic labeling (spec 002 ┬¦6)

The CI submission gate is mechanical ŌĆö schema, determinism, `prov.json`, SF battery
run, intervention-surface detection, one-command rerun. It does **not** decide "is
this really Lux-Marchesi?" or "which mechanism label does this response belong
under?". Those judgments are confined to the **frozen semantic registry**
(`atlas.registry.FROZEN_REGISTRY`), versioned and changed deliberately (spec 002
┬¦12). Operational dependence is not eliminated; it is limited to the semantic layer.

## 5. Failure house (spec 002 ┬¦9)

A model that fails (e.g. price-less Minority Game has no return series) is recorded
as a **structured** failure, not dropped: it needs contract compliance, a declared
claim, a reproducible run bundle, and a failure-taxonomy assignment
(`atlas.profiles.FAILURE_TAXONOMY`). Without these it is an unfinished model, not
negative evidence.

## 6. Reference adapters (first Atlas rows)

The first internal reference atlas (`docs/atlas/reference_atlas_v0.json`) is
generated by dogfooding this contract over the canonical `abm_models`: ZI, CB, LM,
FW, CI, SG (return models ŌåÆ SF profiles) and MG, GCMG (price-less ŌåÆ structured
failure notes). See `atlas.registry.FROZEN_REGISTRY` for families, mechanism
labels, declared channels, and the diversity coverage (spec 002 ┬¦7).

## 7. Not in v1

- Scalar validity rank / leaderboard (spec 002 ┬¦1 non-goal).
- Scoring beyond the six GT-free profile axes (`atlas.profiles`).
- Exposed C1 channels on canonical models (awaits the order-book channel work).
- Multi-asset / continuous-time / agent-side (Shachi) contract (ŌåÆ v2 when needed).
